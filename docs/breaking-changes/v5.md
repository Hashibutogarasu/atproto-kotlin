# Breaking changes — v5.0.0

## Summary

Every published Maven Central artifact has been renamed to drop the
redundant `at-protocol-` prefix. The group `io.github.kikin81.atproto`
is unchanged. **No consumer code changes are required** — this is a
pure coordinate rename. Update the strings in your build file and
you're done.

## What changed

| Surface                          | Before                                            | After                                 |
|----------------------------------|---------------------------------------------------|---------------------------------------|
| Maven coordinate                 | `io.github.kikin81.atproto:at-protocol-runtime:4.x` | `io.github.kikin81.atproto:runtime:5.x` |
| Maven coordinate                 | `io.github.kikin81.atproto:at-protocol-models:4.x`  | `io.github.kikin81.atproto:models:5.x`  |
| Maven coordinate                 | `io.github.kikin81.atproto:at-protocol-oauth:4.x`   | `io.github.kikin81.atproto:oauth:5.x`   |
| Internal Gradle module path      | `:at-protocol-runtime` (etc.)                     | `:runtime` (etc.)                     |
| Repo directory layout            | `at-protocol-runtime/` (etc.)                     | `runtime/` (etc.)                     |

The published package names (`io.github.kikin81.atproto.*`) are
unchanged. `import` statements in consumer code do not need to be
touched.

## How to migrate

### 1. Update your dependency coordinates

**Gradle Kotlin DSL (`build.gradle.kts`):**

```kotlin
dependencies {
    // remove
    implementation("io.github.kikin81.atproto:at-protocol-runtime:4.9.0")
    implementation("io.github.kikin81.atproto:at-protocol-models:4.9.0")
    implementation("io.github.kikin81.atproto:at-protocol-oauth:4.9.0")

    // add
    implementation("io.github.kikin81.atproto:runtime:5.0.0")
    implementation("io.github.kikin81.atproto:models:5.0.0")
    implementation("io.github.kikin81.atproto:oauth:5.0.0")
}
```

**Version catalog (`gradle/libs.versions.toml`):**

```toml
[versions]
atproto = "5.0.0"

[libraries]
# remove: module = "io.github.kikin81.atproto:at-protocol-runtime"
# add:
atproto-runtime = { module = "io.github.kikin81.atproto:runtime", version.ref = "atproto" }
atproto-models  = { module = "io.github.kikin81.atproto:models",  version.ref = "atproto" }
atproto-oauth   = { module = "io.github.kikin81.atproto:oauth",   version.ref = "atproto" }
```

### 2. Sync and rebuild

```bash
./gradlew --refresh-dependencies build
```

That's it. Any API surface you were already using continues to work —
no class renames, no package renames, no signature changes.

## What stays the same

- **Public API**: every class, function, property, and type in the
  5.0.0 artifacts is identical to 4.9.0. `import` statements do not
  change.
- **Package layout**: `io.github.kikin81.atproto.runtime`,
  `io.github.kikin81.atproto.app.bsky.*`, etc. are unchanged.
- **Group ID**: `io.github.kikin81.atproto` — same.
- **Maven Central hosting**: same repo, same promotion behavior, same
  artifact signing, same javadoc/sources JARs.
- **GitHub Packages mirror**: still at
  `https://maven.pkg.github.com/kikin81/atproto-kotlin` under the
  renamed coordinates.
- **Versioning scheme**: semantic versioning continues; future
  non-breaking changes bump minor/patch under the new coordinates.

## Why this change

The pre-5.0.0 coordinates stutter:

```
io.github.kikin81.atproto : at-protocol-runtime
                 ^^^^^^^    ^^^^^^^^^^^^
                 the group already names the project — the artifact
                 repeated it
```

The repo follows the convention used by Square's OkHttp
(`com.squareup.okhttp3:okhttp`), Retrofit
(`com.squareup.retrofit2:retrofit`), and AndroidX
(`androidx.compose.ui:ui`) — the group identifies the library family,
the artifactId names the specific module. Consumer `build.gradle.kts`
files stay short and align visually with the rest of the Kotlin /
Android ecosystem.

The rename also unifies three inconsistent spellings that coexisted in
the repo (`atproto` in the group and packages, `at-protocol` in module
names and artifactIds, `atproto-kotlin` as the repo name) onto a single
`atproto` spelling everywhere — except user-facing prose, which
continues to use the canonical "AT Protocol".

## Notes

### No relocation POMs

This release does **not** ship `<relocation>` stub POMs at the old
coordinates. A consumer pinned to `at-protocol-oauth:4.x` will see a
hard resolution failure on upgrade to 5.x, not a transparent redirect.
This is an intentional simplification given the current (small)
consumer set. If the library later picks up significant external
traction, relocation POMs can be published as an add-on to the 4.x
line.

### 4.x artifacts remain on Maven Central

Existing 4.x artifacts are immutable and stay on Maven Central
indefinitely. They will receive no further updates; all new releases
ship under the new 5.x coordinates. Pin to `5.0.0` or later to pick
up bug fixes, new features, and Lexicon schema updates.

### Gradle command changes for contributors

Contributors running local Gradle commands need to update:

```bash
# before
./gradlew :at-protocol-oauth:test
./gradlew :at-protocol-generator:generateModels
GOLDEN_UPDATE=1 ./gradlew :at-protocol-generator:test --tests '*GoldenFileTest*'

# after
./gradlew :oauth:test
./gradlew :generator:generateModels
GOLDEN_UPDATE=1 ./gradlew :generator:test --tests '*GoldenFileTest*'
```
